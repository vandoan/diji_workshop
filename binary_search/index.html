Binary Search
tl;dr

1) Get the first letter of the name.
2) Get the midway point.
3) Go left or right (binary). (Left!)
    - Still searching?
      - Yes. Loop back to step 2.
      - No. We're done!

Details:

Well, it's not as simple, a little more complex when it comes to JavaScript. Basically, the code needs a starting point and an end point, and it works from that. If the letter is in the first half, go left, else go right. Update the start and end. Repeat.

With normal size data sets, like a list of 100 items or so, the binary sort is not as important. A basic loop would be as fast and as good, as both would take a millisecond or so to execute. What, you couldn't tell!? However, when you're dealing with a hugmongious data set, let's say over 100,000 items, searching for a name in a tome of data, yeah, then it's important.

	loopCounter ++;
	console.log("loop: " + loopCounter);

	if(firstLetter < names[halfway][0]){
		endIndex = halfway;
		halfway = Math.floor((startIndex + endIndex) / 2);
	} else {
		startIndex = halfway;
		halfway = Math.floor((startIndex + endIndex) / 2);
	}

	
	for(let i = startIndex; i < endIndex; i++){ // update the opacity;

	}
}
